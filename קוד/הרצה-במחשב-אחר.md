# איך מריצים את מערכת הארכיון במחשב אחר (מתוך GitHub)

מדריך צעד-אחר-צעד. לא צריך לדעת לתכנת — רק להעתיק ולהדביק פקודות.

> **חשוב להבין מראש:** ב-GitHub שמור רק ה**קוד** של המערכת — לא הקבצים שהעלית ולא המשתמשים.
> כלומר במחשב החדש תקבל מערכת **נקייה וריקה** (כמו התקנה חדשה): תירשם מחדש כמשתמש ראשון (תהפוך אוטומטית ל"בעלים"), והמאגר יתחיל ריק. אם בעתיד תרצה להעביר גם את הקבצים והחשבונות — זה שלב נפרד (העתקת ה"דיסקים" של Docker), תגיד לי ונעשה אותו יחד.

---

## שלב 1 — להתקין Docker Desktop (פעם אחת)
1. היכנס ל-https://www.docker.com/products/docker-desktop והורד **Docker Desktop** למערכת ההפעלה שלך (Windows/Mac).
2. התקן, הפעל את המחשב מחדש אם הוא מבקש, ופתח את Docker Desktop.
3. חכה שהאייקון של Docker (לוויתן) למעלה יהיה יציב ויאמר **Running**. זהו — לא צריך לגעת בו יותר.

## שלב 2 — להוריד את הקוד מ-GitHub
**הדרך הקלה (בלי להתקין כלום):**
1. היכנס לעמוד המאגר: https://github.com/rpg05832/archive-system-backup
2. לחץ על הכפתור הירוק **Code** ← **Download ZIP**.
3. חלץ (Extract) את ה-ZIP אל תיקייה עם **שם באנגלית**, למשל `C:\archive`.
   (חשוב שהנתיב יהיה באנגלית — זה חוסך תקלות.)
   אחרי החילוץ תהיה תיקייה בערך כזו: `C:\archive\archive-system-backup-main\קוד`

## שלב 3 — ליצור קובץ הגדרות (.env)
1. היכנס לתיקייה `קוד` (זו שיש בה את הקובץ `docker-compose.yml`).
2. תמצא שם קובץ בשם `.env.example`. **העתק** אותו, ושנה את שם ההעתק ל-`.env` (בדיוק כך, עם הנקודה בהתחלה).
3. פתח את `.env` עם Notepad ושנה **שני** ערכים לערכים סודיים משלך:
   - `POSTGRES_PASSWORD=` — סיסמה חזקה כלשהי למסד הנתונים.
   - `JWT_SECRET=` — מחרוזת אקראית ארוכה (אפשר סתם לטרוק הרבה תווים).
   את השאר אפשר להשאיר כמו שהוא. שמור וסגור.

## שלב 4 — להריץ את המערכת
1. פתח חלון **PowerShell** בתוך תיקיית `קוד`:
   בסייר הקבצים, בתוך התיקייה `קוד`, לחץ על שורת הכתובת, כתוב `powershell` ו-Enter.
2. הדבק את הפקודה הבאה ו-Enter (זו ההרצה הראשונה — היא בונה הכול ויכולה לקחת כמה דקות):
   ```
   docker compose -p archive up --build -d
   ```
   (ה-`-p archive` נותן למערכת שם קבוע ועוקף בעיה עם שם התיקייה בעברית.)
3. חכה שיסיים. אפשר לראות התקדמות ב-Docker Desktop בלשונית Containers.

## שלב 5 — ליצור את טבלאות מסד הנתונים (פעם אחת בלבד)
במסד נתונים חדש צריך ליצור את הטבלאות. הדבק (באותו חלון PowerShell):
```
docker compose -p archive exec backend npx prisma migrate deploy
docker compose -p archive restart backend
```
- הראשונה יוצרת את כל הטבלאות.
- השנייה מפעילה מחדש את השרת כדי שיזרע את תפקידי ההרשאה.

## שלב 6 — להיכנס למערכת
פתח דפדפן וגלוש אל:
- **האתר:** http://localhost:3000
- הירשם — המשתמש הראשון שנרשם הופך אוטומטית ל**בעלים** (יש לו הכול).

כתובות נוספות (לא חובה):
- בדיקת השרת: http://localhost:4000/health
- צפייה במסד הנתונים: http://localhost:8080 (System=PostgreSQL, Server=postgres, והמשתמש/סיסמה/שם-מסד מתוך `.env`)
- תפיסת מיילים בפיתוח (קודי אימות 2FA): http://localhost:8025

## פקודות שימושיות (להרצה מתוך תיקיית `קוד`)
- **עצירה** (בלי למחוק מידע): `docker compose -p archive down`
- **הפעלה מחדש** אחרי עצירה: `docker compose -p archive up -d`
- **לראות מצב הקופסאות:** `docker compose -p archive ps`
- **לראות לוג של השרת:** `docker compose -p archive logs -f backend`

## אם משהו לא עובד
- ודא ש-Docker Desktop פתוח ומציג **Running** לפני ההרצה.
- אם פורט תפוס (למשל 3000 או 5432) — שנה את המספר המתאים בקובץ `.env` והרץ שוב את שלב 4.
- ההורדה/בנייה הראשונה דורשת אינטרנט (להורדת רכיבי הבסיס). זה חד-פעמי.
- התעודה `netfree-ca.crt` כלולה בקוד וזה תקין גם במחשב שאינו מאחורי NetFree — אין צורך לגעת בה.
